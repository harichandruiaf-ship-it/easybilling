# SOP 06 — Month-end / GST review using Reports

## Goal

Review **billing and tax** figures for a closed period using the **Reports** module.

## Prerequisites

- Invoices for the period are already **generated** in the app (not only drafted offline).

## Steps

1. Go to **Reports** (`#/reports`).
2. Set the **period** to the range you need (e.g. **This month** or **India FY**, or a **custom** from–to range).
3. Review **summary tables and charts** for billed amounts and GST components as shown.
4. If your deployment offers **export**, download CSV/Excel/PDF as needed for your accountant.
5. Cross-check a few large invoices against **Invoice register** filters for the same dates.
6. Record any **adjustments** as new invoices or credit behaviour per your policy (outside this tool if not supported).

## Success criteria

- Report totals reconcile with a sample of invoices for the same window.
- You have an export or screenshot suitable for filing or review.

## Common issues

- **Dashboard vs Reports mismatch:** The **Dashboard** uses a **rolling 36-month** invoice load and optional FY filter; **Reports** uses the **period you pick**. Use Reports for month-end boundaries; see [Appendix A](../APPENDIX_BUSINESS_LOGIC.md).
